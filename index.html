A word counter tallies how many words, characters, and sentences are in a piece of text, and estimates how long it takes an average reader to get through it. Writers use it to hit an assignment's word limit; editors use it to see if a paragraph has run long; students use it to check an essay meets a minimum. This one also strips spaces from the character count separately, since "characters with spaces" and "characters without" are counted differently by most style guides and submission forms.

Reading time here is estimated at roughly 200 words per minute, which is the commonly cited average adult silent-reading speed. It's a rough guide, not a guarantee — dense technical writing reads slower, casual prose reads faster.

02 Case Converter UPPER, lower, Title, Sentence, camelCase, snake_case.

Case converters switch text between the capitalization styles used in writing and in code. Title Case and Sentence case are for headlines and prose. camelCase and snake_case are naming conventions programmers use for variables — camelCase is standard in JavaScript, snake_case is standard in Python. kebab-case is common in URLs and CSS class names, since spaces and underscores aren't always safe there.

Retyping a heading in all caps, or renaming a variable to match a team's style guide, is a small task that's easy to get wrong by hand — a stray capital or a missed underscore breaks code silently. This tool applies the rule consistently across the whole block of text at once.

03 JSON Formatter & Validator Pretty-print, minify, and catch syntax errors.

JSON (JavaScript Object Notation) is the format most web APIs use to send data back and forth — it's built from nested key-value pairs, lists, and a few basic data types. When JSON is generated by a server or squeezed into a log file, it often arrives as one unreadable line. Formatting adds consistent indentation so the structure — which brackets belong to which object — becomes visible at a glance.

The validator half matters just as much as the formatting: a single missing comma or unclosed bracket will make an entire JSON document fail to parse, and the error messages from most systems don't point at the problem clearly. This tool flags exactly where parsing breaks, which is usually faster than reading the raw text character by character. Minify does the reverse of formatting — it strips whitespace, which is useful when you need to paste JSON somewhere with a size limit or ship it in production where every byte counts.

04 Base64 Encode / Decode Convert text to and from Base64.

Base64 is a way of representing binary data — or any text — using only 64 printable characters (letters, digits, plus a couple of symbols). It's not encryption and it isn't meant to keep anything secret; it exists because some systems, like older email formats or certain URL fields and config files, can only reliably handle plain text characters, not arbitrary bytes. Base64 lets images, files, or unusual text get carried safely through those systems.

Common real-world uses: embedding a small image directly inside a CSS or HTML file instead of linking to a separate file, encoding API keys or credentials inside configuration strings, or reading a JSON Web Token (JWT), which stores its payload as Base64 between two dots.

05 URL Encode / Decode Percent-encode strings for safe use in URLs.

URLs can only safely contain a limited set of characters. Anything outside that set — spaces, ampersands, question marks, accented letters, emoji — has to be "percent-encoded" into a %XX format before it can be passed as part of a link or a query string without breaking it. A space becomes %20, an ampersand becomes %26, and so on.

This matters most when a piece of text is going to be dropped into a URL parameter — a search query, a redirect link, a tracking tag — since an unencoded special character can cut the URL short or point somewhere unintended. Decoding does the reverse: it turns a percent-encoded string back into readable text, useful for reading what a long tracking link actually says.

06 Password Generator Random passwords with adjustable length and character sets.

A strong password is long and unpredictable — length matters more than complexity rules. A 16-character password drawn from a large character set is far harder to guess or brute-force than an 8-character one with a forced capital letter and a digit swapped in. This generator uses your browser's built-in cryptographically secure random number generator (crypto.getRandomValues), not a simple pseudo-random function, so the output isn't predictable.

It's worth using a different password for every account and storing them in a password manager rather than memorizing or reusing them — reused passwords are one of the most common ways accounts get compromised after a breach on some unrelated site.

07 Slug Generator Turn titles into clean, URL-friendly slugs.

A slug is the part of a URL that identifies a specific page — in example.com/my-great-blog-post, "my-great-blog-post" is the slug. Slugs are lowercase, use hyphens instead of spaces, and drop punctuation, because that combination is safe in every browser, doesn't need percent-encoding, and reads cleanly to both humans and search engines.

Most blogging and e-commerce platforms generate a slug automatically from a title, but they don't always do it well — long titles get truncated awkwardly, or special characters get left in. This tool applies the standard slug rules directly, so you can paste the exact result into a CMS's URL field.

08 Lorem Ipsum Generator Placeholder text by paragraph, sentence, or word count.

Lorem Ipsum is placeholder text designers and developers drop into a layout before the real copy is ready, so they can judge spacing, line length, and typography without being distracted by actual content. It's derived from a scrambled passage of Latin, which is precisely the point — it looks like real text at a glance without any word standing out or reading as meaningful, so reviewers focus on the design rather than the words.

It's been the industry-standard filler text since long before digital design; it just carried over from print typesetting into web and app mockups.

09 Text Diff Checker Compare two blocks of text line by line.

A diff checker compares two versions of a text and highlights exactly what changed, line by line — additions in one color, removals in another. It's the same idea version control tools like Git use to show what a commit changed, applied here to any two blocks of text: a contract redline, two drafts of an email, or a config file before and after an edit.

This is a line-based diff, which means it compares whole lines rather than individual words — the clearest view when comparing paragraphs, code, or structured text where line breaks carry meaning.

02 — questions worth answering

Frequently asked

Is anything I type saved or sent anywhere?

No. Every tool on this page runs in your browser using JavaScript. Nothing you paste in is uploaded to a server, logged, or stored — closing the tab clears it completely.

Do I need to create an account?

No account, no email, no sign-in. Every tool works immediately, and there's no limit on how many times you can use one.

Does this work on mobile?

Yes — the layout adjusts for smaller screens, and every tool works the same on a phone or tablet browser as it does on desktop.

Why is it called Marginalia?

Marginalia are the notes readers and editors leave in the margins of a page — corrections, quick calculations, shorthand. That's the spirit of these tools: small, fast fixes made in passing, not a destination to sit and work in for an hour.
