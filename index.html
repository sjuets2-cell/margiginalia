A word counter tallies how many words, characters, and sentences are in a piece of text, and estimates how long it takes an average reader to get through it. Writers use it to hit an assignment's word limit; editors use it to see if a paragraph has run long; students use it to check an essay meets a minimum. This one also strips spaces from the character count separately, since "characters with spaces" and "characters without" are counted differently by most style guides and submission forms.

Reading time here is estimated at roughly 200 words per minute, the commonly cited average adult silent-reading speed. It's a rough guide, not a guarantee — dense technical writing reads slower, casual prose reads faster.

02 Case Converter UPPER, lower, Title, Sentence, camelCase, snake_case.

Case converters switch text between the capitalization styles used in writing and in code. Title Case and Sentence case are for headlines and prose. camelCase and snake_case are naming conventions programmers use for variables — camelCase is standard in JavaScript, snake_case is standard in Python. kebab-case is common in URLs and CSS class names, since spaces and underscores aren't always safe there.

Retyping a heading in all caps, or renaming a variable to match a team's style guide, is a small task that's easy to get wrong by hand — a stray capital or a missed underscore breaks code silently. This tool applies the rule consistently across the whole block of text at once.

03 JSON Formatter & Validator Pretty-print, minify, and catch syntax errors.

JSON (JavaScript Object Notation) is the format most web APIs use to send data back and forth — built from nested key-value pairs, lists, and a few basic data types. When JSON is generated by a server or squeezed into a log file, it often arrives as one unreadable line. Formatting adds consistent indentation so the structure — which brackets belong to which object — becomes visible at a glance.

The validator half matters just as much: a single missing comma or unclosed bracket makes an entire document fail to parse, and most systems don't point at the problem clearly. This tool flags exactly where parsing breaks. Minify does the reverse — strips whitespace, useful when shipping JSON to production where every byte counts.

04 Base64 Encode / Decode Convert text to and from Base64.

Base64 represents binary data — or any text — using only 64 printable characters. It's not encryption and isn't meant to keep anything secret; it exists because some systems, like older email formats or certain URL fields and config files, can only reliably handle plain text, not arbitrary bytes. Base64 lets images, files, or unusual text get carried safely through those systems.

Common real-world uses: embedding a small image directly inside a CSS or HTML file, encoding API keys inside configuration strings, or reading a JSON Web Token (JWT), which stores its payload as Base64 between two dots.

05 URL Encode / Decode Percent-encode strings for safe use in URLs.

URLs can only safely contain a limited set of characters. Anything outside that set — spaces, ampersands, question marks, accented letters, emoji — has to be "percent-encoded" into a %XX format before it can be passed as part of a link or query string without breaking it. A space becomes %20, an ampersand becomes %26.

This matters most when text is going into a URL parameter — a search query, a redirect link, a tracking tag — since an unencoded special character can cut a URL short. Decoding does the reverse, useful for reading what a long tracking link actually says.

06 Password Generator Random passwords with adjustable length and character sets.

A strong password is long and unpredictable — length matters more than complexity rules. A 16-character password drawn from a large character set is far harder to brute-force than an 8-character one with a forced capital and a swapped digit. This generator uses the browser's built-in cryptographically secure random number generator (crypto.getRandomValues), not a simple pseudo-random function.

Use a different password for every account and store them in a password manager rather than reusing or memorizing them — reused passwords are one of the most common ways accounts get compromised after an unrelated breach.

07 Slug Generator Turn titles into clean, URL-friendly slugs.

A slug is the part of a URL that identifies a specific page — in example.com/my-great-blog-post, "my-great-blog-post" is the slug. Slugs are lowercase, use hyphens instead of spaces, and drop punctuation, since that combination is safe everywhere and reads cleanly to both humans and search engines.

Most CMS platforms generate a slug automatically, but not always well — long titles get truncated awkwardly, special characters get left in. This tool applies the standard rules directly.

08 Lorem Ipsum Generator Placeholder text by paragraph count.

Lorem Ipsum is placeholder text designers drop into a layout before real copy is ready, so they can judge spacing and typography without being distracted by actual content. It's derived from scrambled Latin, which is the point — it looks like real text without any word standing out as meaningful.

It's been the industry-standard filler text since long before digital design — it carried over from print typesetting into web and app mockups.

09 Regex Tester Test a regular expression against sample text and see every match highlighted.

A regular expression (regex) is a compact pattern language for finding or validating text — matching an email format, pulling every number out of a paragraph, checking a password meets a rule. Writing one correctly on the first try is rare; testing it against real sample text is how you actually confirm it does what you meant.

This uses standard JavaScript regex syntax. The g flag (global) finds every match instead of stopping at the first one; i makes it case-insensitive; m changes how ^ and $ behave across multiple lines. Matches are highlighted directly in the text below so you can see exactly what the pattern is catching — and, just as importantly, what it's missing.

10 Color Converter Convert between HEX, RGB, and HSL with a live swatch.

The same color can be written three common ways on the web. HEX packs red, green, and blue into a six-digit code, the most common format in CSS and design tools. RGB spells those same three channels out as numbers 0–255. HSL — hue, saturation, lightness — describes color the way people tend to think about it, which makes it easier to adjust: keep the hue, nudge the lightness, and you've made the same color darker or lighter without guessing new numbers.

Paste any of the three formats in and the other two update automatically, along with a live swatch so you can confirm it's actually the shade you meant.

11 Unix Timestamp Converter Convert between Unix time and a readable date, in either direction.

Unix time counts the seconds elapsed since midnight UTC on January 1, 1970 — a single number that avoids ambiguity about time zones or calendar formats, which is why it's the standard way computers store timestamps internally: in databases, log files, and API responses.

The tradeoff is that a raw Unix timestamp like 1752192000 means nothing to a person at a glance. This tool converts it to a readable date in both your local time zone and UTC, and works in reverse — pick a date and get back the timestamp a system expects.

12 Text Diff Checker Compare two blocks of text line by line.

A diff checker compares two versions of a text and highlights exactly what changed, line by line — additions in one color, removals in another. It's the same idea version control tools like Git use to show what a commit changed, applied here to any two blocks of text: a contract redline, two drafts of an email, a config file before and after an edit.

This is a line-based diff — it compares whole lines rather than individual words, the clearest view when comparing paragraphs, code, or structured text where line breaks carry meaning.

02 — questions worth answering

Frequently asked

Is anything I type saved or sent anywhere?

No. Every tool on this page runs in your browser using JavaScript. Nothing you paste in is uploaded to a server, logged, or stored — closing the tab clears it completely.

Do I need to create an account?

No account, no email, no sign-in. Every tool works immediately, and there's no limit on how many times you can use one.

Does this work on mobile?

Yes — the layout adjusts for smaller screens, and every tool works the same on a phone or tablet browser as it does on desktop.

Why is it called Marginalia?

Marginalia are the notes readers and editors leave in the margins of a page — corrections, quick calculations, shorthand. That's the spirit of these tools: small, fast fixes made in passing, not a destination to sit and work in for an hour.
